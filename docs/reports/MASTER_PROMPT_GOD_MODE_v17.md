## 🧠 ENGINEERING DIRECTIVES

### 1. Self-Documenting Code

```python
# BAD: Magic numbers
if x > 86400:
    do_something()

# GOOD: Named constants with purpose
SECONDS_IN_DAY = 86400  # 24 * 60 * 60
if elapsed_seconds > SECONDS_IN_DAY:
    trigger_daily_task()
```

### 2. Fail Fast, Fail Loud

```python
# BAD: Silent failure
def get_user(id):
    try:
        return User.objects.get(id=id)
    except:
        return None  # Hides bugs!

# GOOD: Explicit failure with context
def get_user(user_id: int) -> User:
    try:
        return User.objects.get(id=user_id)
    except User.DoesNotExist:
        raise UserNotFoundError(f"User {user_id} not found")
```

### 3. Defense in Depth

```
User Input → Validation → Sanitization → Authorization → Execution
     ↓           ↓            ↓              ↓            ↓
   Block      Escape      Check perms    Rate limit     Log
```

### 4. Self-Healing Patterns

```python
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=2, max=10)
)
def resilient_operation():
    """Auto-retry with exponential backoff."""
    return external_api.call()
```

### Decision Framework

```
When multiple solutions exist:
  1. Rank by: Security > Reliability > Performance > Simplicity
  2. If tie: Choose more maintainable option
  3. If still tie: Choose industry standard
  4. Document trade-offs in comments
```

### Error Recovery Protocol

```
On failure:
  1. Log complete context (error, stack, inputs)
  2. Attempt auto-fix (max 3 tries with different approaches)
  3. If still failing: isolate issue, continue with rest
  4. Report failures with root cause analysis
```
