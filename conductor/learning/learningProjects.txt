## PART 1: ADVANCED BACKEND ARCHITECTURE (The "Conductor")

### 1.1 Domain-Driven Design (DDD) & Service Layers
Move beyond "Fat Models" and "Fat Views".
- **Entity**: The core business object (e.g., `Course`, `User`).
- **Service**: The business logic coordinator (e.g., `enroll_user`, `calculate_xp`).
- **Signal**: The decoupling agent. `UserService` emits `UserEnrolled`, `GamificationService` listens.

**Pattern**:
```python
# signals.py
user_enrolled = Signal()

# services.py
def enroll(user, course):
    Enrollment.objects.create(user=user, course=course)
    user_enrolled.send(sender=Course, user=user)

# receivers.py (Gamification)
@receiver(user_enrolled)
def award_enrollment_xp(sender, user, **kwargs):
    XPService.award(user, 50)
```

### 1.2 Event-Driven Architecture (EDA)
Decouple systems in time and space.
- **Sync**: Direct function calls (tight coupling).
- **Async**: Celery Tasks (e.g., Send Email).
- **Event**: Signals (in-process) or Redis Pub/Sub (distributed).

### 1.3 God-Tier Security (Zero Trust)
- **CSP**: Content Security Policy. Whitelist script sources.
- **JWT Rotation**: Access Token (5m), Refresh Token (7d). Rotation prevents replay attacks.
- **Throttling**: Leaky Bucket algorithm via Redis. 100 req/min/IP.

## PART 2: FRONTEND ENGINE (The "Interface")

### 2.1 The Flutter Render Pipeline
Understand how pixels hit the screen.
1.  **Build**: Widget Tree (Configuration).
2.  **Layout**: Render Object Tree (Sizing/Positioning).
3.  **Paint**: Layer Tree (Drawing commands).
4.  **Composit**: GPU instructions (Skia/Impeller).

**Optimization**: Use `RepaintBoundary` to isolate frequent repaints (e.g., a progress bar) from static content.

### 2.2 Riverpod Architecture

## PART 5: MENTAL MODELS

### 5.1 The CAP Theorem
In a distributed system, you can only pick 2:
- **Consistency** (Read = Write)
- **Availability** (Always respond)
- **Partition Tolerance** (Survive network cut)
*Reality*: P is mandatory. You trade C vs A. `PostgreSQL` = C. `Cassandra` = A.

### 5.2 Amdahl's Law
Max speedup is limited by the serial part of the program.
$S(s) = \frac{1}{(1-p) + \frac{p}{s}}$
*Lesson*: Don't optimize parallel parts if the serial bottleneck dominates.
