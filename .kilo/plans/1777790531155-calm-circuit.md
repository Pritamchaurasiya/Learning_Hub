## 🔴 PHASE 2: CRITICAL ISSUES FOUND

### 2.1 Environment Configuration Mismatch ⚠️ CRITICAL

**Issue**: Inconsistent API URLs across env files:

| File | VITE_API_URL |
|------|--------------|
| `.env.example` | `http://localhost:5000/api/v1` |
| `.env.development` | `http://localhost:8000/api/v1` |

**Impact**: Frontend may call wrong backend port in development (8000 vs 5000)

**Fix Required**: Standardize to port 5000 (Express backend):
```bash
# .env.development me update karo:
VITE_API_URL=http://localhost:5000/api/v1
```

### 2.2 Backend Auth Middleware Missing File ⚠️ MODERATE

**Issue**: `backend/src/middleware/authMiddleware.ts` referenced but not found. Authentication is actually in `backend/src/utils/auth.ts` and routes use directly.

**Current**: `backend/src/routes/index.ts:56` imports `authenticate` from `../utils/auth`

**Status**: Works correctly, but file naming is misleading. Consider renaming or documenting.

### 2.3 QuizService Type Mismatch ⚠️ MODERATE

**Issue**: `QuizQuestion` interface in `quizService.ts:16-24` defines:
```typescript
correct_answer: string  // string in API
```

But `useStore.ts:44` defines quiz answer mapping as:
```typescript
answers: Record<string, number> // questionId -> optionIndex (number!)
```

**Impact**: When submitting quiz, answer indices (numbers) may not match backend expected string answers, causing scoring errors.

**Fix**: Align types — either frontend stores string option values, or backend converts index to string.

### 2.4 AdminPage Hardcoded Mock Data ⚠️ HIGH

**Issue**: `AdminPage.tsx:372-401` hardcodes mock course/student data instead of calling real APIs.

**Impact**: Admin dashboard shows fake data, not actual system state.

**Fix**: Replace mock data with actual API calls from `adminService.getUsers()` and `adminService.getCourses()` (functions already exist but not used in some tabs).

### 2.5 Missing Input Sanitization in Some Endpoints ⚠️ HIGH

**Issue**: Backend validations exist (`backend/src/validations/schemas.ts`), but some endpoints skip validation middleware:

Check routes:
- `/auth/register` — ✅ validated
- `/auth/login` — ✅ validated
- `/auth/refresh` — ✅ validated
- `/courses/enroll` — ✅ validated
- `/tests/:id/submit` — ✅ validated
- `/users/bookmarks` — ✅ validated

**Status**: Mostly good, but verify all POST/PUT routes have `validate(schema)`.
