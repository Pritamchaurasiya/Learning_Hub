## 2. Core Concepts

### 2.1 State Management with Riverpod

**What it is**: Riverpod manages app state in a reactive, testable way.

**Why we use it**:
- Compile-time safety (no runtime errors like Provider)
- Easy testing with overrides
- Automatic dependency disposal

**Example**:
```dart
// Provider definition
final userProvider = StateNotifierProvider<UserNotifier, User?>((ref) {
  return UserNotifier(ref.read(apiClientProvider));
});

// Usage in widget
class ProfileScreen extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final user = ref.watch(userProvider);
    return Text(user?.displayName ?? 'Guest');
  }
}
```

**Common Mistakes**:
- ❌ Using `ref.read` inside `build()` - causes missed updates
- ✅ Use `ref.watch` for reactive reading in widgets

---

### 2.2 API Client Architecture

**What it is**: Centralized HTTP client with security, retry, and caching.

**Why we need it**:
- Single point for authentication headers
- Automatic token refresh on 401
- Retry logic for flaky networks
- Request signing for security

**Key Features**:
```dart
// Retry with exponential backoff
await _executeWithRetry(() => _dio.get<dynamic>(path));

// Automatic token refresh
if (error.response?.statusCode == 401) {
  final refreshed = await _refreshToken();
  if (refreshed) {
    // Retry with new token
  }
}
```

**Security Best Practices**:
- Store tokens in `flutter_secure_storage` (encrypted)
- Never log tokens or sensitive data
- Use HTTPS only
- Validate SSL certificates

---

### 2.3 Type Safety in Dart

**What it is**: Ensuring all types are explicitly declared, avoiding `dynamic`.

**Why it matters**:
- Catches bugs at compile time, not runtime
- Better IDE autocomplete
- Clearer code intent

**Before (Unsafe)**:
```dart
final response = await _dio.post(path);  // Returns Response<dynamic>
final data = response.data['key'];       // No type checking
```

**After (Type-Safe)**:
```dart
final response = await _dio.post<Map<String, dynamic>>(path);
final data = response.data!['key'] as String;  // Explicit type
```

## 3. Windows Desktop Development

### 3.1 Prerequisites

1. **Flutter SDK** - `flutter doctor` should show all green
2. **Visual Studio 2022** with:
   - Desktop development with C++ workload
   - MSVC v143 build tools
   - Windows 10/11 SDK

### 3.2 Build Commands

```powershell
# Debug build
flutter run -d windows

# Release build
flutter build windows --release

# Output location
build/windows/x64/runner/Release/learning_hub.exe
```

### 3.3 Common Issues

| Issue | Solution |
|-------|----------|
| `No CMAKE_CXX_COMPILER` | Install VS C++ Desktop workload |
| `Unable to generate build files` | Delete `build/windows` and retry |
| Plugin linker errors | Run `flutter clean` then rebuild |

## 5. Security Practices

### 5.1 Input Validation

**Always validate user input**:
```dart
bool _isValidResourceId(String id) {
  final regex = RegExp(r'^[a-zA-Z0-9_-]+$');
  return regex.hasMatch(id) && id.length <= 100;
}
```

### 5.2 Secure Storage

**Never store tokens in SharedPreferences**:
```dart
// ❌ Bad - tokens visible in plain text
await prefs.setString('token', token);

// ✅ Good - encrypted storage
await FlutterSecureStorage().write(key: 'token', value: token);
```

### 5.3 SSL Pinning

Prevent man-in-the-middle attacks:
```dart
dio.interceptors.add(CertificatePinningInterceptor(
  allowedSHA256Fingerprints: ['7a:12:f3:84:cc...'],
));
```

## 6. Performance Optimization

### 6.1 Widget Optimization

```dart
// ❌ Bad - rebuilds entire tree
ListView.builder(
  itemBuilder: (_, i) => ExpensiveWidget(data[i]),
)

// ✅ Good - uses const and keys
ListView.builder(
  itemBuilder: (_, i) => ExpensiveWidget(key: ValueKey(data[i].id), data: data[i]),
)
```

### 6.2 Image Caching

Use `cached_network_image` for network images:
```dart
CachedNetworkImage(
  imageUrl: course.thumbnailUrl,
  placeholder: (_, __) => const Shimmer(),
  errorWidget: (_, __, ___) => const Icon(Icons.error),
)
```
