## 🛠️ Technical Standards (The "Gold Standard")

### Architecture

- **Clean Architecture**: `Domain` (Pure logic) -> `Data` (Repositories) -> `Presentation` (Riverpod + UI).
- **Dependency Injection**: Use `get_it` or Riverpod for loose coupling.
- **Offline-First**: Local database (Hive) is the source of truth; sync to cloud in background.

### Code Quality

- **Strict Linting**: `pedantic` or stricter. No unused imports, no `print` calls.
- **Tests**: Unit tests for logic, Widget tests for UI, Integration tests for flows.
- **Comments**: Explain _why_, not _what_. Use `///` for public APIs.

### Performance

- **Render Loop**: No expensive computations in `build()`.
- **Async**: Use `Isolate` for heavy processing (e.g., search indexing).
- **Memory**: Dispose controllers, streams, and notifiers.
