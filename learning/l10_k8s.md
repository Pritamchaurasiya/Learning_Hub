### 🐳 Beyond Docker Compose

Docker Compose is great for a single server. But what if you need:

- **Zero-Downtime Deployments**?
- **Auto-Scaling** (10 servers when busy, 1 when quiet)?
- **Self-Healing** (Restart crashed containers)?

### ☸️ Kubernetes (K8s)

K8s is an "Orchestrator". It manages your containers across a cluster of servers.

**Key Concepts**:

1.  **Pod**: The smallest unit. Runs one or more containers (e.g., your Django App).
2.  **Deployment**: Manages Pod replicas (e.g., "Keep 3 Pods running").
3.  **Service**: Load Balancer. Directs traffic to Pods.
4.  **Ingress**: The entry point (e.g., Nginx) handling domains and SSL.

### ⛑️ Helm: The Package Manager

Writing K8s YAML files is tedious. **Helm** is like `pip` or `npm` for K8s.

- **Chart**: A package of pre-configured K8s resources.
- **Action**: `helm install my-app ./charts/django-app`.
